# Hocus Pocus Access Flow Setup (Supabase)

## Required environment variables

- `ADMIN_SECRET`: secret used by `admin.html` for approve/reject API calls.
- `SUPABASE_URL`: project URL (for example `https://xyzcompany.supabase.co`).
- `SUPABASE_SERVICE_ROLE_KEY`: service role key (server-side only).

Optional table name overrides:

- `SUPABASE_APPLICATIONS_TABLE` (default: `applications`)
- `SUPABASE_MEMBERS_TABLE` (default: `members`)
- `SUPABASE_INVITES_TABLE` (default: `invites`)

## SQL schema

Run this in Supabase SQL editor:

```sql
create table if not exists applications (
  id bigint generated by default as identity primary key,
  email text not null,
  linkedin text not null,
  status text not null default 'pending',
  "createdAt" timestamptz not null default now(),
  source text not null default 'public',
  "inviteToken" text,
  "approvedAt" timestamptz,
  "rejectedAt" timestamptz,
  "memberId" bigint,
  "memberToken" text
);

create table if not exists members (
  id bigint generated by default as identity primary key,
  email text not null,
  linkedin text not null,
  "accessToken" text not null unique,
  "createdAt" timestamptz not null default now(),
  "inviterMemberId" bigint,
  "applicationId" bigint
);

create table if not exists invites (
  id bigint generated by default as identity primary key,
  token text not null unique,
  "memberId" bigint not null,
  "createdAt" timestamptz not null default now(),
  "expiresAt" timestamptz not null,
  "usedAt" timestamptz,
  "claimedByMemberId" bigint
);

create index if not exists applications_email_idx on applications (email);
create index if not exists members_access_token_idx on members ("accessToken");
create index if not exists invites_member_id_idx on invites ("memberId");
create index if not exists invites_token_idx on invites (token);
```

## URLs

- Public landing: `/`
- Private event page: `/inside.html?token=<memberToken>`
- Invite claim: `/inside.html?invite=<inviteToken>`
- Admin: `/admin.html`

## Notes

- Invite links are one-time and expire after 48 hours.
- Each member can generate a maximum of 2 invite links.
- If Supabase variables are missing, APIs fall back to in-memory storage (local testing only, non-persistent).
